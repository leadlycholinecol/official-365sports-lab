# official-365sports-lab

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常为静态内容，不依赖特定后端服务，可直接在浏览器中打开浏览。

## 项目简介

官方实验室将各类独立页面统一归档于此，便于版本管理、分发与维护。仓库中的每个 HTML 文件均为独立单元，互不依赖，内容涵盖实验性界面、信息展示页、工具页面等。

## 目录说明

```
/
├── README.md           # 本文件
├── index.html          # 可选：仓库首页/索引页
├── pages/              # 独立 HTML 页面目录（推荐）
│   ├── page1.html
│   └── page2.html
└── assets/             # 公共资源（如 CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/`：存放所有独立 HTML 页面，每个文件为一个完整页面。
- `assets/`：存放页面共享的静态资源文件，如图片、样式表、脚本等。
- `index.html`：可选，用于展示仓库的导航或总览。

## 页面归档说明

- 每个 HTML 文件均独立完整，可直接在浏览器中打开使用。
- 页面不依赖特定域名或外部不可控服务，可离线或本地运行。
- 归档内容保持原始结构与功能，仅进行必要的版本标识或注释更新。
- 历史版本通过 Git 提交记录保留，不单独保留旧文件。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有公共资源一同放入 `assets/`。
- 更新页面：直接修改对应文件，并在提交信息中注明变更内容。
- 删除页面：移除文件并确保无其他页面引用，提交时注明原因。
- 本仓库不接收外部直接推送，如有建议或问题请通过 Issues 反馈。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接双击 `pages/` 中的 HTML 文件，或使用本地服务器（如 `python -m http.server`）浏览。
3. 如仓库包含 `index.html`，可将其作为入口页面。
